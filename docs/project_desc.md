# Data Mining Project Guidelines

## Purpose
This project is part of the continuous assessment for the Data Mining course and counts for 20% of the final grade.

The goal is to apply course concepts to a real dataset through a clear, well-structured data mining workflow. The final work should show that the team can:
- understand the dataset and its context,
- define a relevant analytical objective,
- clean and prepare the data,
- apply suitable data mining methods,
- interpret results correctly,
- communicate findings clearly.

## Work Format
- Group project of up to three students.
- All members should contribute to the analysis, report writing, and presentation.

## Expected Focus
The project should emphasize data mining rather than only descriptive statistics.

Depending on the dataset and objective, the project may focus on:
- prediction,
- classification,
- clustering,
- anomaly or pattern detection,
- dimensionality reduction,
- extracting business-relevant insights.

You do not need to use every possible method. Choose methods that fit the dataset and the question well.

## Software
- Python or R may be used.

## Suggested Report Structure

### 1. Title Page
Include:
- project title,
- student names,
- course title,
- video presentation link.

### 2. Dataset Description
Explain:
- the context of the data,
- the source,
- who collected it and when,
- what one observation represents,
- the main variables.

### 3. Problem Statement
Clearly define:
- the problem being studied,
- the objective of the analysis,
- the main research question(s),
- the target variable if relevant.

### 4. Data Preparation
Document and justify preprocessing choices such as:
- missing-value handling,
- outlier treatment,
- variable transformations,
- categorical encoding,
- scaling or normalization,
- feature selection.

### 5. Methods and Analysis
Present the methods used and explain why they fit the problem.

Possible examples:
- Decision Trees
- Random Forests
- KNN
- Text Mining
- Naive Bayes
- Logistic Regression
- Clustering
- PCA
- Association Rules
- Neural Networks

Code should be organized and explained, not pasted without context.

### 6. Results and Interpretation
Include:
- graphs, tables, and model outputs,
- relevant performance metrics,
- interpretation of the main results,
- discussion of key findings.

The emphasis should be on what the results mean, not only on displaying them.

### 7. Conclusion
Summarize:
- the project objective,
- the methods used,
- the most important results,
- the practical insights from the analysis.

## Evaluation Criteria

| Criterion | Points |
| --- | ---: |
| Clarity and relevance of the problem statement | 10 |
| Dataset understanding and data preparation | 20 |
| Appropriateness of methods and technical implementation | 20 |
| Quality of results, evaluation, and interpretation | 20 |
| Report presentation and conclusion | 10 |
| Presentation and oral discussion | 20 |
| Total | 100 |

## Presentation Expectations
- Maximum duration: 8 minutes per group
- Language: English
- Timing: last two weeks of April 2026
- A live demonstration of the model is expected during the presentation

Suggested flow:
- 1-2 minutes: dataset and project objective
- 2 minutes: data preparation and method
- 2 minutes: results and interpretation
- 1-2 minutes: live model run
- 1 minute: conclusion

All group members should participate and come prepared with the files needed for a smooth demo.

## Important Remarks
- The project must reflect your own work.
- The analysis should be clear, serious, and well structured.
- Superficial work or copied content may be penalized.
- A strong project is not the one using the most methods, but the one using the right methods well and explaining them clearly.
